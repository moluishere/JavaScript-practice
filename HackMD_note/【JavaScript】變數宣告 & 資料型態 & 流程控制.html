變數 variable

  • 為什麼需要變數？

    • 變數的功能是用來輸入、處理及儲存外界的資料，而變數在使用以前均要事先宣告才可使用。可以想像變數為存放資料的一個標籤，我們將資料指定給變數，就可以透過呼叫變數來使用資料。
    • 想的簡單一點，我們需要給資料一個名字，才能去使用那個資料。

  • 命名原則

    • 可使用英文字母或數字，但第一個字母不可以用數字
    • 不能使用保留字
    • 注意大小寫

  • 宣告 vs 未宣告

    • 未宣告：ReferenceError: age is not defined

      console.log(age);

    • 已宣告未定義：undefined

      var age;
console.log(age);

  • 變數提升

    • Var 變數宣告會有提升
    • 使用 var 宣告
    


    console.log(age); //undefined 
var age = 123;
console.log(age); //123
    • 使用 let 宣告
    




    console.log(age);
let age = 123;
console.log(age);
// Uncaught ReferenceError: Cannot access 'age' before initialization

常數 const

  • 常數不能再指定（re-assign）

    const AGE = 123;
AGE = 456;
console.log(AGE);
// TypeError: Assignment to constant variable.

  • 再指定不是指值的更改，而是盒子（記憶體地址）的更改

    • 指向的盒子沒有更改，順利執行：
    let x = [];
x.push(1, 2, 3, 4);
console.log(x);
// 印出 [ 1, 2, 3, 4 ]

    let y = { name: "Mary", age: 30 };
y.name = "Peter";
console.log(y);
// 印出{ name: 'Peter', age: 30 }
    • 指向的盒子re-asign，噴錯：
    const x = [];
x = [1,2,3,4]
//Uncaught TypeError: Assignment to constant variable.

    const y = {name: 'Mary', age: 30}
y = {name: 'Peter'};
//Uncaught TypeError: Assignment to constant variable.

  • 修改 vs 賦值

    • 下面的例子，第 7 行 a 修改了值（記憶體／盒子沒變），第 10 行的 b 指向的是修改了值的 a
    









    var a = [1, 2, 3];
var b = a;

console.log(a); // [ 1, 2, 3 ]
console.log(b); // [ 1, 2, 3 ]

a[0] = 10;

console.log(a); // [ 10, 2, 3 ]
console.log(b); // [ 10, 2, 3 ]
    • 下面的例子，第 7 行 a 已經重新賦值（記憶體／盒子被更改了），b 指向的仍是一開始宣告的 a
    










    var a = 123;
var b = a;

console.log(a); // 123
console.log(b); // 123

a = 456;

console.log(a); // 123
console.log(b); // 456

資料運算

a++ 與 ++a







let a = 1;
a += 1; // a = a + 1 
// 可簡寫成 a++ （先丟a再遞增）
// ++a（先遞增再丟a）

console.log(a); // 2





let a = 1;

console.log(a++); // 1 ; 這行的 a++ 先把 a 丟出去
console.log(a); // 2 ; 這行的 a 遞增






let a = 1;

console.log(a++); // 1
console.log(++a); // 3
console.log(a); // 3






let a = 1;

a = ++a + a++ + a;

console.log(a); // 7

字串運算





let username = "Yvonne";
console.log("My name is" + " " + username + " " + ",and I am 18 years old");
console.log(`My name is ${username}, and I am 18 years old`);
// My name is Yvonne, and I am 18 years old

真假值／布林值 Boolean

  • 只有兩種結果：true or false

不同型別之間的轉換

  • 型別轉換的函式

    





    let a = 123; // 123
let b = String(123); // "123"
let c = Number("456"); // 456
let d = Boolean(0); // false 

console.log(a, b, c, d);

    • 在 JavaScript 中會被轉為 false 的值有：

      • "" 空字串
      • 0,-0, NaN
      • null
      • undefined
      • false

    • 而除了以上的值之外，都會被轉為 true，舉例如下

      • "Hello World" 非空字串
      • 42 非零的有效數字
      • [], [1, 2, 3] 陣列，不管是不是空的
      • {}, { name: 'Jack' } 物件，不管是不是空的
      • function foo() {} 函式
      • true

  • NaN

    


    let a = Number("ccc");
console.log(a); // NaN = Not a number
console.log(typeof a); // number
    • NaN 本身也是 number 的一員，但它不等於任何人，包括它自己
    • NaN 的可以使用內建的 isNaN() 函數來判斷

  • 強迫轉型 Type Coercion

    在 JavaScript 中的資料型別是動態的，若沒有給定變數的型別，JavaScript 會進行判斷，而強制轉型，例如有用到運算子，JavaScript 就會覺得你是要讓值變成數字進行運算，因此就會將運算子左右的值強制轉型為數字。
    JavaScript 中的強制轉型（coercion）分為兩種，分別是「顯性轉型」（explicit coercion）及「隱性轉型」（implicit coercion）。

流程控制 判斷式

if...else

if ( 條件式 ){
  // 某條件成立會執行這個 { } 區塊中的內容
  // ...
}
else{
  // 否則執行 else 這個 { } 區塊中的內容
  // else 可以有也可以沒有
}

let age = 20;

if (age >= 18) {
  console.log("已成年");
}

let age = 20;

if (age >= 18) {
  console.log("已成年");
} else {
  console.log("未成年");
}

// 印出 已成年

三元運算子

  • 下述例子的 2 行的程式碼，等同於上面例子 5 行的程式碼
  • 注意程式碼的可讀性，if...else 相對容易閱讀，除非你很習慣三元運算子的寫法，不然還是選if…else`
let age = 20;
age >= 18 ? console.log("已成年") : console.log("未成年");
// 印出 已成年

  // 某條件成立會執行這個 { } 區塊中的內容
  // ...
}
else if( 條件式2 ){
  // ...
}
else if( 條件式3 ){
  // ...

else if( 條件式4 ){
  // ...
}
else{
  // 若上面所有條件判斷都是 false ，才會執行 else 這個 { } 區塊中的內容

}










let age = 16;

if (age >= 18) {
  console.log("已成年");
} else if (age >= 12 && age <= 18) {
  console.log("青少年");
} else {
  console.log("未成年");
}

Switch

switch 的用法就是要告訴你，符合哪樣就請跑出相對應結果。
再用 case 去分類其結果，並在尾巴加上 break，表示一個結果的結束，但還有下一種可能。在視覺上比較有架構的整理，當流程很多的時候，用有架構整理會較易閱讀。

















let gender = 100;

switch (gender) {
  case 1:
    console.log("男");
    break;

  case 2:
    console.log("女");
    break;

  default:
    console.log("不想說");
    break;
}
// 印出 不想說

流程控制 迴圈

for 迴圈

  • 用法： for ( 起始條件; 終止條件; 每一圈的改變 ) { 每一圈要做的事情 }

  • 範例：




for ( var i=1; i<=100; i++) {
    console.log(i)
}
  • 練習題：
    起始條件是 i = 1，終止條件是 i <= 100，每一圈的改變是 i = i+1
    符合條件就會執行 { } 裡面的事情。








// 印出 1 ~ 10
for (var a = 1; a <= 10; a++) {
  console.log(a);
}

for (var a = 0; a < 10; a++) {
  console.log(a + 1);
}






// 印出 20 以內的 雙數，倒著印

for (let a = 20; a >= 0; a = a - 2) {
  console.log(a);
}

while 迴圈








// whilt 的條件滿足時，執行 { } 裡面的動作。

var i = 1
while (i <= 100) {
    console.log(i)
    i++
}
  1. 一開始宣告一個變數 i = 1
  2. 判斷是否符合 while 的條件，i = 1 有符合條件
  3. 執行 { } 裡面的動作：console.log 印出 i 之後，再把 i + 1
  4. 再判斷一次是否符合 while 的條件，i = 2 有符合條件
  5. 再繼續執行 { } 裡面的動作
  6. 直到不符合條件為止
  • 補充：
    • continue 可以跳過這圈未完成的動作，直接進到下一圈迴圈
    • break 可以強制跳出迴圈
